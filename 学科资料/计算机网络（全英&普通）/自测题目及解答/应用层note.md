## 一、 DNS 域名系统：互联网的“导航仪”

DNS 是大题背景和选择题的高频点，重点在于查询方式、协议选择及现代安全技术。

### 1. 域名结构与关键参数

- **标签与级别**：域名从右向左看，级别依次降低。根域在最右边（通常省略点）。
    
- **区 (Zone) vs 域 (Domain)**：区是 DNS 服务器实际管辖的范围。一个域可以被划分为多个区。
    
- **FQDN (绝对域名)**：末尾带点的完整路径，全球唯一。
    
- **关键参数**：
    
    - **域名长度**：最长为 **255** 个字符。
        
    - **解析起点**：主机解析时先查 **hosts.txt**。
        

### 2. 解析方式（必考对比）

- **递归查询 (Recursive)**：“一查到底”。主机 $\rightarrow$ 本地域名服务器。本地域名服务器代劳询问，直接返还结果。
    
- **迭代查询 (Iterative)**：“指路模式”。本地域名服务器 $\rightarrow$ 根/顶级/权威服务器。对方告诉本地域名服务器“下一个该去问谁”。
    
- **高速缓存与 TTL**：缓存不具权威性。TTL 减到 0 记录失效。更新 DNS 记录未立即生效是因为旧缓存未到 TTL。
    

### 3. 协议选择与安全加速

- **默认使用 UDP (端口 53)**：追求速度。
    
- **切换 TCP 的两种情况**：
    
    1. **数据太大**：UDP 报文超过 512 字节（TC 标志位置位）。
        
    2. **区域传输**：主从服务器同步数据（要求可靠）。
        
- **负载均衡与加速**：
    
    - **CDN/AnyCast**：完成就近解析，在不损失速度的前提下实现负载均衡。
        
- **安全加密**：
    
    - **DoT / DoH**：针对原生 DNS 明文传输的不安全特性，使用 TLS/HTTPS 加密 DNS 请求。

## 二、 UDP 的重要应用（RPC & RTP）

考试常考这两个协议为何选择 UDP：

- **RPC (远程过程调用)**：
    
    - **核心功能**：允许本地程序调用远程主机上的过程，隐藏底层通信细节。
        
    - **特点**：追求高效率，通过简单的请求/应答完成任务。
        
- **RTP (实时传输协议)**：
    
    - **核心功能**：应用于网络电话、视频、网游等实时多媒体。
        
    - **关键点**：将多个实时数据复用到一个 UDP 流中，支持单播和组播。

## 三、 万维网与 HTTP：数据的“搬运工”

考点集中在 RTT 计算、连接类型、状态管理及高效处理技术。

### 1. HTTP 的演进与 RTT（计算题常客）

- **非持续连接 (HTTP 1.0)**：每个对象建立一次 TCP 连接。耗时：**2 RTT + 文档传输时间**。
    
- **持续连接 (HTTP 1.1)**：
    
    - **非流水线**：发一个，等一个响应，再发下一个。
        
    - **流水线**：一口气发一堆请求，不等响应就接着发（效率最高）。
        
- **HTTP/3 延迟**：配合 **TLS 1.3**，理论上可以实现近乎 **0-RTT** 的访问延迟。
    

### 2. Cookie、代理与高效处理

- **Cookie**：HTTP 本身是无状态的。服务器通过 `Set-Cookie` 让浏览器记住用户身份。
    
- **代理服务器 (Web Cache)**：减少重复请求源服务器，节省带宽，提高响应速度。
    
- **TCP 移交 (TCP Splicing/Handoff)**：
    
    - **原理**：TCP 端点被直接传递给后端处理节点。
        
    - **好处**：处理完成后直接返回给用户，无需再次经过负载均衡器，实现高效负载均衡。
